Technical Detail

Facts for Nerds.
How it works under the hood.

Architecture
Playwright-core → Browserless remote Chromium via CDP (connectOverCDP) — no local browser, serverless-safe
axe-core fetched from CDN and injected via addScriptTag post-render — runs after hydration, not on source
Deterministic scanning + probabilistic AI synthesis — two separate layers
Next.js API route on Node runtime (force-dynamic) — Edge runtime cannot run Playwright
Reliability
Stable load strategy: waitForLoadState("networkidle") + 1500ms buffer — ensures SPAs finish rendering before scan
SSRF protection layer — blocks localhost, 127.0.0.1, and private network ranges before any request is made
Partial data tolerance — report generates even if one signal is incomplete
Top violations capped at 10 — signal-to-noise optimised for actionability, not exhaustiveness
Data structure
axe-core violations normalised to: { id, impact, description, help, helpUrl, nodes }
UX insights stored as structured schema: { finding, risk, suggestion, signal }
Detection and explanation are separate — axe flags, AI explains and prioritises independently
Feedback layer
Each insight rated at the insight level — not the whole report
Ratings stored per insight type and per product context
Dataset accumulates for prompt refinement and signal weighting — not theoretical, already running
What's next technically
Pattern detection
Surface recurring issues across multiple pages and audit runs automatically
Regression comparison
Diff quality scores between runs on the same URL — catch what changed
Benchmark dataset
Normalised quality baselines per product area — track drift over time
